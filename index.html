Análisis del archivo de prueba utilizando Autopsy

Luego de eso se inicio el programa de Autopsy en Kali Linux ejecutando el comando autopsy con permisos de administrador desde linea de comandos.

Lo primero que se debe de realizar es crear un Nuevo Caso en autopsy

# Nombre del Caso
CasoDePruebas
# Descripción
Pruebas forences para tarea 8
# Investigadores
Gari Mendoza

Lo siguiente que se debe de realizar es agregar un nuevo Host

Para el Host lo dejaremos por defecto utilizando el siguiente nombre

# Host Name
host1

Cuando ya tenemos creado el Host debemos de agregar una nueva imagen que será la que se va a analizar

Para esto debemos de agregar una nueva imagen dando clic en la opción de “Add Image File”

Se debe de ingresar la locación donde se encuentra el disco que se va a analizar que seria “/home/lester/ext-part-test-2.dd”, se debe seleccionar que es un disco y seleccionaremos el método de importación “Move”

Lo primero que podemos ver al cargar el disco para ser analizado es que detecto varias particiones que serian las siguientes:

Particiones
Partition 1 - C:
Partition 2 - D:
Partition 3 - E:
Partition 4 - F:
Partition 5 - G:
Partition 6 - H:

Para este caso seleccionamos el disco E: y daremos clic en la opción de Detalles y lo primero que debemos de hacer antes de analizar la partición es cargar los String’s para esto lo haremos utilizando los botones Extract Strings y Extract Unallocated

Cuando ha terminado de cargar los Strings entonces tendremos algo como solo siguiente:

El siguiente paso que se debe de realizar es analizar la partición en donde completamos los strings para eso seleccionamos la partición y damos clic en el botón de analizar

Lo primero que vamos a revisar será el File Analysis donde podremos ver los archivos que se encuentren en esa partición, para este caso encontramos un archivo .txt

También se tiene la opción para ver los archivos borrados al analizar en la pestaña de All Delete Files

Esto nos sirve para ver todos los documentos que hayan sido borrados en la partición.
Luego se tiene la opción de File Type en esta opción lista todos los archivos separados por el tipo de archivo
